# Security Policy

## Supported Versions

只对最新一个 minor 版本和上一个 minor 版本提供安全更新。

| Version | Supported |
|---------|-----------|
| 0.2.x   | ✅ |
| 0.1.x   | ⚠️ 仅严重漏洞 |
| < 0.1   | ❌ |

## Reporting a Vulnerability

发现安全问题请**不要**公开提 issue。请走以下流程：

1. **私下邮件报告**：发邮件给项目作者邮箱（在 commit 历史的 `Author` 字段里可找到）
   - 标题前缀：`[ClaudeBackup Security]`
   - 内容：影响版本、复现步骤、潜在影响范围
2. **48 小时内回复**确认收到并开始分析
3. **7 天内**给出修复方案（或说明为何不修）
4. **30 天后**才会公开披露漏洞细节，给用户升级时间

## 关于本应用的安全模型

`ClaudeBackup` 是**纯本地工具**，但涉及几类敏感操作，请用户和审计者了解风险面：

### 不存储凭证
- **GitHub push** 走系统 Git Credential Manager — 凭证由 Windows 凭据管理器保管，
  本应用不读不存。
- **NAS 路径**只是文件系统路径，不带账号密码字段。

### 网络流量
- 仅在用户**主动点「发布版本」**时通过 `git push` 走 HTTPS 到 GitHub
- 不发送任何 telemetry / 崩溃上报 / 检查更新请求
- 不连任何 ClaudeBackup 作者控制的服务器

### 本地文件系统访问
- 读：用户选定的项目目录 + 备份位置
- 写：备份位置（git-backups / git-bundles / snapshots）+ 用户配置目录 `~/.claude-backup/`
- 不写其他位置，不修改项目源码（除非用户主动点「恢复版本」）

### 系统集成
- **资源管理器右键菜单**：HKCU 用户级注册表，无管理员权限
- **登录自启**：用户启动文件夹放 `.lnk`，无管理员权限
- **Task Scheduler**：用户级任务（`-RunLevel Limited`），无管理员权限
- 安装包本身**需要管理员权限**写 `Program Files`，但运行时一律以普通用户身份跑

### 第三方依赖
| 依赖 | 用途 | 漏洞风险 |
|---|---|---|
| PySide6 (Qt) | GUI | 极低（成熟 LGPL 项目） |
| git for Windows | 备份引擎（CLI 调用） | 跟随 git 自身 |
| Pillow | 图标生成（仅构建期） | 不进运行时 |
| PyInstaller | 打包（构建期） | 不进运行时 |

## 已知非问题

以下场景是设计如此，不会按安全漏洞处理：

- 用户的备份目录权限设置过宽，第三方进程能读 — 用户自己决定备份位置，自己负责权限
- `~/.claude-backup/config.json` 是明文 — 里面没有任何凭证，只是路径偏好
- 错误对话框显示文件路径 — 故意为之，方便排查
- 日志里包含项目名 / 备份路径 — 故意为之，方便排查
